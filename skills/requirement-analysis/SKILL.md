# 需求与分析阶段

你正在执行 Implement 流程的**第 1 阶段：需求与分析**。

## 执行步骤

1. 从需求描述中提取功能点和业务规则。
2. 对每条规则做四要素结构化：
   - 规则描述
   - 满足时行为
   - 违反时行为
   - 边界条件
3. 主动挑战前提假设（Premise Challenge）：
   - 这是对的问题吗？
   - 如果什么都不做会怎样？
   - 现有代码中有什么可复用的？
4. 产出 `requirement-analysis.md`，包含：
   - 功能点清单
   - 业务规则（四要素格式）
   - 风险初评

## 门禁

产出 `requirement-analysis.md` 后，**必须请人确认分析结果**，确认后才能进入下一阶段（spec）。

## 注意

- 不要在这个阶段写代码。
- 不要跳过前提挑战直接进入功能点提取。
- 规则穷举策略：CRUD 补全 / 正反面 / 边界值 / 异常流 / 交叉影响。
