# 方案设计阶段

你正在执行 Implement 流程的**第 3 阶段：方案设计**。

本阶段回答"怎么做"——先做代码搜证和澄清，再把 `spec.md` 转化为**可批次执行**的 `tasks.md`。

## 前置条件

`spec.md` 已完成且人已确认。

## 核心升级

- `tasks.md` 不再只是任务清单，也兼作 coding 阶段的**轻量状态文件**。
- 代码搜证优先交给子代理 `design-clarifier`（或等价的 `Task / Subagent` 调用）做上下文隔离。
- 只把真正阻塞的问题抛给人，其余默认决策沉淀到"自动决策记录"。

## 执行步骤

### 1. 读取输入与现状

1. 读取 `spec.md`，确认目标、接口定义、文件规划、测试策略和约束。
2. 如存在，读取 `.codebuddy/context/current-task.md`，补充上下文和风险信息。
3. 读取相关代码、契约、Rules 和可复用模板，优先确认现有实现，不要先假设需要新增文件。

### 2. 生成澄清草稿

1. 优先调用子代理 `design-clarifier`；如果运行环境不支持显式点名子代理，则用 CodeBuddy 原生 `Task / Subagent` 能力按同等职责隔离执行。
2. 子代理只负责代码搜证和分级澄清，输出 `.codebuddy/context/clarifications.draft.md`。
3. 澄清草稿必须遵循 `.codebuddy/templates/clarifications-draft-template.md`，按 `P0 / P1 / P2` 输出：
   - **P0**：阻塞性问题，必须请人确认。
   - **P1**：优先通过代码验证后仍不能 100% 确认的问题。
   - **P2**：自动决策记录，只展示，不打断当前流程。

### 3. 处理澄清并收敛设计边界

1. 读取 `.codebuddy/context/clarifications.draft.md`。
2. 若存在 `P0` 或无法默认处理的 `P1`，向人提问并 **STOP**，等待确认后再继续。
3. 若人回复"你定 / 暂不确定"，采用草稿中的默认处理，并把该决策写进 `tasks.md` 的"自动决策记录"。
4. `P2` 不向人追问，直接沉淀为设计阶段的默认决策。

### 4. 生成批次化 `tasks.md`

1. 使用 `.codebuddy/templates/tasks-batch-template.md` 作为结构模板。
2. `tasks.md` 必须包含：
   - `## 用户前置任务（如有）`
   - `## 自动决策记录`
   - `## 批次总览`
   - `## 批次详情`
3. 批次划分原则：
   - **契约与高风险依赖优先**，放在最前面的批次。
   - 每个批次必须能被**单独审查、单独验证**，避免把过多文件揉成一个批次。
   - 小需求优先 1 个批次；中等需求优先 2-3 个批次；复杂需求一般不超过 5 个批次。
   - 批次顺序必须体现依赖关系，后批次不能依赖未完成的前批次实现。
4. 每个任务必须包含：
   - 顶层任务勾选项：`- [ ]`
   - **文件**：具体要改哪个文件
   - **内容**：具体实现内容
   - **DoD（完成定义）**：明确做到什么算完成
   - **验证**：怎么验证该任务完成
   - **回退**：如果做砸了怎么退
5. `DoD` 里使用普通列表，不嵌套任务勾选框，避免 coding 阶段误判状态。
6. 所有任务必须可追溯到 `spec.md` 的某一项，不允许"与任务 N 类似"这类模糊引用。
7. 在所有批次详情之后，增加 `## 任务依赖关系` 部分，用 ASCII DAG 格式标注任务间的前后依赖和可并行关系（参考模板中的示例格式）。当前用于辅助人审查和串行执行时的排序判断。

### 5. 按需生成 `design-notes.md`

当需求存在高风险决策、复杂依赖或临时折中时，再补 `design-notes.md`，用于承载关键技术决策和风险评估。

## 门禁

- 产出 `tasks.md` 后，**必须请人审查任务拆解、批次边界和自动决策**，确认后才能进入下一阶段（coding）。
- 一旦进入 coding，除非明确回流到 design，否则**不再重排已确认批次**，仅允许更新任务勾选状态。

## 注意

- 本阶段不写实现代码，只做搜证、澄清和任务拆解。
- `tasks.md` 中禁止 `TBD`、`TODO` 等占位符。
- 任务清单必须让 coding 阶段可以直接逐批执行，而不是再临时二次拆解。
