# 方案设计阶段

你正在执行 Implement 流程的**第 3 阶段：方案设计**。

本阶段回答"怎么做"——将 spec 转化为可执行的任务清单。

## 前置条件

`spec.md` 已完成且人已确认。

## 执行步骤

1. 读取 `spec.md`，确认目标、接口定义、文件规划、测试策略和约束。
2. 读取现有代码和契约，梳理系统现状（通过代码和 MCP）。
3. 生成任务清单 `tasks.md`：
   - §0 契约定义：更新 `openapi.yaml`（如有接口变更）
   - §1+ 按 spec 中的文件规划，逐文件拆任务
   - 每个任务包含：具体文件路径、要做什么、验证步骤
   - 任务粒度尽量小（参考 2-5 分钟一个任务）
4. 按需生成 `design-notes.md`（关键技术决策与风险评估）。

## 门禁

产出任务清单后，**必须请人审查任务拆解**，确认后才能进入下一阶段（coding）。

## 注意

- 本阶段不写实现代码，只拆任务。
- 任务清单中禁止"TBD""TODO"等占位符。
- 任务不能写"与任务 N 类似"，必须展开具体内容。
- 所有任务必须可追溯到 `spec.md` 中的某一项。
