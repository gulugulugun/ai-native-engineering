# 测试与验证阶段

你正在执行 Implement 流程的**第 5 阶段：测试与验证**。

## 前置条件

编码阶段所有任务已完成且验证通过。

## 执行步骤

1. 基于 `requirement-analysis.md` 中的业务规则生成测试用例：
   - 满足时 → 正向用例
   - 违反时 → 反向用例 + 错误码验证
   - 边界条件 → 边界值测试
2. 产出 `test-cases.md`（结构化测试用例清单）。
3. 执行验证：
   - lint 检查
   - 编译检查
   - 契约校验（`openapi.yaml` 与实际接口是否一致）
4. 每次声称测试通过必须附带命令输出证据（退出码 + 通过/失败数）。

## 门禁

**必须请人验收最终结果**，确认关键场景覆盖后才能进入下一阶段（archiving）。

## 注意

- 不接受"应该能过""之前跑过了"等推断。
- 测试用例应覆盖所有四要素结构化规则，不遗漏。
